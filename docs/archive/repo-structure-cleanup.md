## Decisions

### D1: Do not merge tools/ and scripts/

**Trade-off:** Merging would unify utility scripts under one directory but
requires updating 24+ cross-references across docs, CI workflows
(release.yml), CODEOWNERS, starters, and tests. The two directories serve
genuinely different purposes (dev utilities vs CI/verification).

**Decision:** Document the boundary via READMEs and the AGENTS.md table.

**Principle:** Minimize blast radius — prefer documentation over structural
churn when the cost/benefit ratio is unfavourable.

### D2: Do not restructure src/

**Trade-off:** Renaming phase directories to semantic names (e.g. validators/,
phases/) would improve discoverability but break all test imports and
documentation examples.

**Decision:** Add src/README.md. The phase naming maps to a deliberate
conceptual model with full test coverage.

**Principle:** Prefer the boring option — existing names are well-tested
and understood.

### D3: Starter template over creating .github/copilot-instructions.md

**Trade-off:** Creating the file directly in this repo would close the
"referenced but missing" gap immediately. However, `.github/copilot-instructions.md`
is a platform entry point for ADOPTER projects. This repo is the substrate.

**Decision:** Provide `starters/copilot-instructions.md`. Adopters copy it.

**Principle:** agent-enterprise ships zero project-specific content (AGENTS.md
contract + command-centre/ anchoring constraint).

### D4: Preserve root-relative backtick path convention

**Trade-off:** Updating CONTEXT.md table paths from `skills/<name>/` to
`../skills/<name>/` would be technically correct relative paths from docs/.
But every existing docs/ file uses root-relative backtick paths for
structural references.

**Decision:** Keep root-relative backtick paths. Update only line 8's
`` `docs/SKILL_AUTHORING_GUIDE.md` `` to `` `SKILL_AUTHORING_GUIDE.md` `` because
referencing `docs/X` from within docs/ is genuinely confusing.

**Principle:** Consistency over correctness when the convention is
well-established and understood.

### D5: CHANGELOG entries are immutable

**Trade-off:** Updating historical CHANGELOG path references would keep
them "correct" but would falsify the historical record.

**Decision:** Leave CHANGELOG.md historical entries untouched.

**Principle:** Append-only logs are immutable.

### D6: Conditional AGENTS.md pointer in starter

**Trade-off:** Unconditionally pointing to AGENTS.md assumes all adopter
projects have one. Many won't — they use copilot-instructions.md as their
primary agent context.

**Decision:** Instructional comment guides adopters: point to AGENTS.md if
it exists, otherwise this file is the primary context.

**Principle:** Starters serve all adopter types without assuming they've
replicated agent-enterprise's full structure.
